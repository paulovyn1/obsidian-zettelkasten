# Estrutura de Vault Recomendada — Método Triwer

> Usada no onboarding (O3) quando o vault é novo ou desorganizado. Propor ao usuário e
> criar somente com aprovação. Pastas marcadas como *(opcional)* só criar se fizerem
> sentido para o trabalho do usuário.

```
00. 🎲 Processamento/          → efêmeras brutas, inbox de ideias
01. ⚓ Âncoras/                → MOCs temáticos
02. 📆 Daily Notes/            → diário (opcional)
03. 📽️ Projetos/               → projetos ativos e produtos
  └── Conteúdo Gerado/         → output de lives, aulas, eventos (opcional)
  └── Lançamentos/             → histórico e planejamento de lançamentos (opcional)
  └── [Produto X]/             → uma pasta por produto registrado na memória
04. 🧩 Recursos/Notas/         → corpo principal do Zettelkasten
  └── 1. Efêmeras/             → captura rápida (fleeting)
  └── 2. Referência/           → artigos, citações, livros, estudos, teorias
  └── 3. Literatura/           → síntese de fontes externas
  └── 4. Permanentes/          → insights próprios, evergreen
  └── 6. Estratégias/          → estratégias concretas destiladas
  └── 7. Autores/              → notas por autor (opcional)
Modelos/                       → templates (opcional)
```

## Mapeamento para a memória

Ao criar esta estrutura, registrar em `memoria.md`:

| Papel | Pasta |
|---|---|
| Efêmeras/inbox | `04. 🧩 Recursos/Notas/1. Efêmeras/` (captura) e `00. 🎲 Processamento/` (inbox bruta) |
| Referência | `04. 🧩 Recursos/Notas/2. Referência/` |
| Literatura | `04. 🧩 Recursos/Notas/3. Literatura/` |
| Permanentes | `04. 🧩 Recursos/Notas/4. Permanentes/` |
| Estratégias | `04. 🧩 Recursos/Notas/6. Estratégias/` |
| Âncoras/MOCs | `01. ⚓ Âncoras/` |
| Projetos | `03. 📽️ Projetos/` |
| Daily notes | `02. 📆 Daily Notes/` |
| Templates | `Modelos/` |

## Âncoras iniciais sugeridas

Sugerir conforme os temas do usuário (O4) — exemplos comuns para quem trabalha com
conteúdo e vendas: `Marketing` · `vendas` · `Lançamento` · `Estratégia` · `Narrativas` ·
`AI` · `Gestão` · `Produtividade` · `estudos`

Criar apenas as que o usuário aprovar, usando o template `moc`.
